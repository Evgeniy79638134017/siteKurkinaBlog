# Основные инструкции для Gemini — Проект SiteKurkina

Сайт Ольги Куркиной — нутрициолога и эксперта по биохакингу для 50+. Целевая аудитория — женщины 50+, желающие вернуть энергию и здоровье.

## Текущее состояние (на 24 марта 2026)
- **Архитектура:** Готова (маршруты, компоненты, контент-данные).
- **Визуал:** Сгенерировано 5 из 10 фото, требуется интеграция в код. Квота на генерацию исчерпана до 31 марта. Полное исследование — в `visual_strategy.md`.
- **UI-компоненты:** shadcn/ui установлен, стилизация настроена, но компоненты в папку `ui/` пока не добавлены. Отсутствуют формы.
- **Подробный аудит:** См. файл `audit.md`.

## Стек технологий
- **Фреймворк:** Next.js 16 (App Router, Server Components, TypeScript strict). Все команды (`pnpm dev` и др.) выполнять ИЗ ПАПКИ `site/`.
- **Стилизация:** Tailwind CSS v4. **КРИТИЧНО: CSS-first подход.** НИКАКОГО `tailwind.config.js`. Вся тема и дизайн-система заданы в `src/app/globals.css` через директиву `@theme {}`.
- **UI Библиотека:** shadcn/ui (стиль new-york). Убедитесь, что `components.json` имеет пустое поле `tailwind.config`.
- **Анимации:** Motion 12 (обязательно импорт из `motion/react`, а **не** `framer-motion`). В CSS используется `tw-animate-css` вместо `tailwindcss-animate`. Используйте `cn()` из `@/lib/utils`.
- **Иконки:** Lucide React.
- **Формы:** React Hook Form + Zod, отправка email через Resend (пока не реализовано). Используйте `sonner` для тостов.

## Дизайн-система (см. globals.css)
- **Аудитория 50+:** Крупный шрифт (базовый 18px), высокий контраст, простая навигация, отсутствие визуального мусора.
- **Фон:** cream `#F7F0E6`, beige `#F4E1C4`.
- **Акценты:** sage `#96AB88`, terra `#D08C60`.
- **Текст:** dark `#2C1810`, muted `#6B5B52`.
- **Шрифты:** Cormorant Garamond (заголовки) + DM Sans (текст).

## Структура проекта (`site/src/`)
- `app/` — Страницы (App Router).
- `components/layout/` — Header, Footer, FloatingCTA.
- `components/sections/` — Крупные блоки (HeroSection, TrustBar и т.д.).
- `components/shared/` — Мелкие UI/обертки (AnimatedSection).
- `components/ui/` — Компоненты shadcn (кнопки, аккордеоны — *добавлять по мере необходимости*).
- `lib/content/` — Данные TypeScript (`cases.ts`, `programs.ts`, `quotes.ts`).
- `lib/constants.ts` — Контакты, ссылки, конфиг.

## Правила написания кода
1. **Named Exports:** Использовать везде (кроме `page.tsx`/`layout.tsx`).
2. **Server Components:** По умолчанию. Директива `"use client"` — только там, где нужны хуки состояния или `motion`.
3. **Mobile-First:** Сначала стилизуем для мобильных (`sm:`, `md:`, `lg:`).
4. **Компоненты Next:** Обязательно `next/image` для фото и `next/link` для внутренних ссылок.
5. **Контент:** Весь текст вынесен в `src/lib/content/` или берется из `ContentFiles/`. *Не хардкодить большие блоки текста в JSX.*
6. **Фото:** Располагаются в `public/images/`.

## Алгоритм работы (воркфлоу)
- **Только абсолютные пути** при работе с файлами.
- Если нужно создать новый функционал: сначала продумать **ПЛАН**, затем писать код. Делать маленькие изменения и проверять.
- Внимательно читать контекст предыдущих шагов (в `audit.md` и `task.md` в папке `.gemini/antigravity/brain/...`).
